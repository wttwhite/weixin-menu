# Meal Plans Inventory Shopping Generation Design

更新日期：2026-04-21

## 目标

增强 `pages/meal-plans/index` 的食材库存检查弹框：

- 弹框更宽，更接近用户参考图
- 缺货食材带勾选框，默认全选
- `生成采购清单` 不再是占位按钮，而是根据勾选的缺货项真正新建采购清单
- 生成成功后直接跳转到采购页查看结果

## 范围

- 修改用餐计划页库存检查弹框的展示与交互
- 缺货项勾选状态的页面内管理
- 生成采购清单按钮文案与数量联动
- 把勾选的缺货项写入一张新采购清单
- 生成成功后跳转到采购页
- 补充对应测试

## 非目标

- 不改动库存检查的食材匹配逻辑
- 不把有库存项加入可勾选范围
- 不复用已有采购清单，本轮每次都新建采购清单
- 不在本轮支持“选择目标采购清单”
- 不在库存检查弹框内编辑采购项详情

## 交互设计

### 弹框布局

- 弹框面板加宽，使用更接近参考图的居中宽版样式
- 保留现有标题、日期、统计卡和食材列表
- 统计卡继续展示：
  - 需食材
  - 库存充足
  - 需采购

### 缺货项勾选

- 只有 `缺货` 项显示勾选框
- `有库存` 项不显示勾选框
- 弹框打开时，所有缺货项默认选中
- 点击勾选框可切换选中状态
- 勾选状态只影响采购清单生成，不影响库存检查结果本身

### 生成采购清单按钮

- 按钮文案改为 `生成采购清单 (N)`
- `N` 取当前被勾选的缺货项数量
- 若没有任何缺货项被勾选：
  - 不提交
  - 提示用户先选择要采购的食材

### 成功结果

- 生成成功后跳转到 `/pages/shopping/index`
- 不继续停留在库存检查弹框中
- 回到计划页后：
  - 当前月份
  - 当前选中日期
  - 日历展开状态
  - 当天计划列表
  保持原样

## 数据设计

### 库存检查项扩展

在页面层为 `inventoryItems` 增加只读派生字段：

- `selectable`
  - `缺货` 为 `true`
  - `有库存` 为 `false`
- `selected`
  - 仅缺货项可为 `true/false`
- `checkboxClass`
  - 用于区分选中/未选中的视觉状态

页面额外维护：

- `inventorySelectedKeys`
  - 当前被选中的缺货项 key 列表

### 采购清单生成

每次点击生成都创建一张**新的**采购清单。

建议写入形态：

- 采购清单名称：
  - `${inventoryCheckDateLabel} 食材补货`
  - 例如：`2026-04-06 食材补货`
- 采购清单日期：
  - 使用库存检查对应日期
- 采购项：
  - 只根据当前勾选的缺货项生成
  - 名称：缺货食材名
  - 数量/单位：来自库存检查结果里的 required 数量
  - 来源：`manual`
  - 备注：可附带来源说明，例如“来自 2026-04-06 库存检查”

## 复用策略

- 库存检查结果的计算继续复用现有 `buildInventoryCheckResult(...)`
- 采购清单写入优先复用现有 shopping service / shopping page 数据结构
- 采购项生成逻辑在计划页内做最小映射，不重写购物域模型

## 错误处理

- 若库存检查还在加载：
  - 不允许生成采购清单
- 若没有勾选任何缺货项：
  - toast 提示
- 若生成采购清单失败：
  - 保持当前弹框打开
  - 保留当前勾选状态
  - toast 提示错误原因
- 若生成成功但跳转失败：
  - 至少 toast 提示“已生成采购清单”
