# Pantry Manager Modal Unification Design

更新日期：2026-04-23

## 目标

统一食材分类与食材位置的管理弹框实现，去掉库存页“库存配置”里的位置管理，并让“我的”页的食材分类/食材位置复用同一套库存管理弹框设计与交互。

## 范围

- 库存页“库存配置”弹框只保留食材分类管理
- 从库存页现有分类/位置管理 UI 中抽离一套共享食材管理弹框组件
- “我的”页的食材分类入口改为复用该共享弹框
- “我的”页的食材位置入口改为复用该共享弹框，并沿用库存配置里位置管理的视觉结构
- 为“我的”页补齐食材分类/食材位置的排序交互，使其与库存页共享同一套列表式管理体验
- 更新对应模板测试与页面流测试

## 非目标

- 不改动菜谱分类弹框
- 不调整库存新增/编辑表单中的分类和位置选择方式
- 不新增新的食材管理能力范围之外的设置项

## 现状

- 库存页当前把食材分类和食材位置都内嵌在“库存配置”弹框中
- “我的”页当前存在两套实现：
  - 菜谱分类使用独立组件 `recipe-category-manager-modal`
  - 食材分类/食材位置使用 `profile` 页内联的 `manager-modal`
- 两套食材管理弹框的视觉和结构不一致，后续维护会继续分叉

## 设计

### 1. 共享食材管理弹框组件

- 新增一个共享的食材管理弹框组件，直接承接当前库存配置里分类/位置管理的列表式设计
- 组件负责：
  - 标题区
  - 输入框 + 确定按钮
  - 加载态与空态
  - 列表、数量徽标、编辑/删除按钮
  - 拖拽把手与拖拽事件透传
- 组件不直接发起云调用，仍由页面层传入数据并处理事件

### 2. 库存页

- 保留“库存配置”入口，但弹框内容只剩“食材分类”
- 不再在库存页中展示“位置管理”区块
- 库存页继续保留分类新增、重命名、删除、拖拽排序
- 库存页仍保留位置数据加载能力，用于库存新增表单与详情编辑表单，不作为当前弹框 UI 展示的一部分

### 3. 我的页

- 保留三个入口：
  - 菜谱分类
  - 食材分类
  - 食材位置
- 菜谱分类保持现有组件
- 食材分类与食材位置统一改为打开共享食材管理弹框
- 食材位置的标题、输入 placeholder、空态文案与数量单位改为位置语义，但结构与库存配置中的位置管理一致
- 旧的 `manager-modal` 结构与样式删除，避免继续维护两套实现

### 4. 数据与交互

- 页面层继续根据 mode 决定调用：
  - `listPantryCategories` / `listPantryLocations`
  - `createPantryCategory` / `createPantryLocation`
  - `updatePantryCategory` / `updatePantryLocation`
  - `deletePantryCategory` / `deletePantryLocation`
  - `reorderPantryCategories` / `reorderPantryLocations`
- “我的”页新增对食材分类和食材位置的拖拽排序持久化
- 删除按钮继续只在 `pantryItemCount === 0` 时展示

## 测试策略

- 模板测试：
  - 库存页不再渲染位置管理区块
  - “我的”页不再渲染旧 `manager-modal`
  - “我的”页引入共享食材管理弹框组件
- 流程测试：
  - 库存页打开配置弹框时只加载食材分类
  - “我的”页食材分类与食材位置入口都能打开共享弹框并加载对应数据
  - “我的”页支持食材位置（以及食材分类）的重命名/排序交互
